파일 1: app.py  (Flask 서버 — 가장 핵심 파일)
================================================================

역할: 이 프로젝트의 두뇌. ESP32에서 데이터를 받고,
      낙상 판단을 하고, 웹 대시보드를 서비스합니다.

실행 방법: python app.py  (포트 5000)

─── 주요 전역 변수 ───────────────────────────────────────────

  latest_data     - ESP32에서 마지막으로 받은 센서 데이터
                    height, angle, mmwave, door, fall_candidate,
                    fall_detected, servo_status, timestamp 포함

  occupancy{}     - 입퇴실 상태
                    state: "empty" / "occupied" / "long_stay"
                    occupied: True / False

  system_sm{}     - 12단계 AI 상태 머신 현재 상태
                    state: EMPTY / OCCUPIED / FALL_DETECTED 등

  response_state{}  - 낙상 발생 후 60초 응답 대기 상태
                      mode: "idle" / "awaiting" / "responded" / "notified"

  audio_state{}   - DFPlayer Mini 오디오 재생 상태
                    playing, current_track, mode 포함

  connection_status{} - 하드웨어 연결 상태 (ESP32, Cloudflare URL 등)

  audio_cmd_queue - ESP32가 폴링해서 가져가는 오디오 명령 큐

─── 주요 함수 ────────────────────────────────────────────────

  save_data(data)
    - ESP32에서 POST로 받은 데이터를 처리하는 핵심 함수
    - 낙상 감지 시 자동으로 오디오 큐에 명령 추가
    - 상태 머신 전환 실행

  compute_system_state()
    - 현재 센서 데이터 기반으로 12가지 상태 중 하나 반환
    - 우선순위: RECOVERY > RESPONSE_RECEIVED > GUARDIAN_ALERTED
              > AWAITING_RESPONSE > FALL_DETECTED > FALL_CANDIDATE
              > HEIGHT_DROP > MONITORING > OCCUPIED > DOOR_OPEN > EMPTY

  compute_fall_confidence()
    - 낙상 신뢰도 0~100% 계산
    - height, mmwave, door, fall 플래그 조합

  update_occupancy(data)
    - 문 열림/닫힘 + mmWave 조합으로 입퇴실 판단
    - BL0303 자석 스위치 기반

  _full_exit_reset(data)
    - 퇴실 시 모든 낙상/오디오 상태 초기화

─── API 엔드포인트 전체 목록 ──────────────────────────────────

  [인증 불필요 — ESP32 내부망용]
  POST /data                  - ESP32 센서 데이터 수신 (핵심)
  GET  /esp32/audio-command   - ESP32가 오디오 명령 폴링
  POST /button/press          - 아케이드 버튼 눌림 이벤트
  GET  /esp32/config          - ESP32 시작 시 설정 가져가기
  GET  /api/info              - API 목록 요약
  GET/POST /api/tunnel-url    - Cloudflare Tunnel URL 등록/조회

  [인증 필요 — 보호자 대시보드용]
  GET  /                      - 메인 대시보드 페이지
  GET  /settings              - 설정 페이지
  GET  /dev-console           - 개발자 콘솔 페이지
  GET  /api/latest            - 최신 센서+상태 JSON (1초마다 폴링)
  GET  /api/logs              - 최근 20개 센서 로그
  GET  /api/events            - 실시간 이벤트 로그 (문개폐, 낙상 등)
  GET  /api/connection-status - 하드웨어 연결 상태
  GET/POST /api/settings      - 서버 설정 읽기/저장
  GET  /api/raw-data          - 전체 서버 상태 스냅샷 (디버그용)
  POST /start-scan            - 환경 스캔 시작
  GET  /scan-status           - 스캔 진행 상태
  GET  /scan-data             - 스캔 결과 데이터
  POST /audio/play_music      - 배경음악 재생 (0001.mp3)
  POST /audio/stop            - 오디오 정지
  POST /audio/warning         - 경고음 재생 (0002.mp3)
  POST /audio/voice           - 음성 재생 (0003.mp3)
  POST /audio/volume          - 볼륨 조절

  [시뮬레이션 테스트 — ESP32 없이 테스트할 때]
  POST /test/normal           - 정상 상태 주입
  POST /test/warning          - 낙상 의심 주입
  POST /test/height_drop      - 높이 급감 주입
  POST /test/fall             - 낙상 감지 주입
  POST /test/enter            - 입실 시뮬레이션
  POST /test/exit             - 퇴실 시뮬레이션
  POST /test/long_stay        - 장시간 체류 시뮬레이션
  POST /test/respond          - 낙상 응답 버튼 시뮬레이션

    "height": 165.0,

─── 12가지 상태 머신 정의 ──────────────────────────────────────

  상태명              │ 의미            │ 단계
  ────────────────────┼─────────────────┼──────────
  EMPTY               │ 욕실 비어있음   │ 정상
  DOOR_OPEN           │ 문 열림 (빈 욕실)│ 정상
  ENTERING            │ 입실 중         │ 정상
  OCCUPIED            │ 욕실 사용 중    │ 정상
  MONITORING          │ 케어 모드 (장시간)│ 정상
  HEIGHT_DROP         │ 높이 급감 감지  │ 경계
  FALL_CANDIDATE      │ 낙상 의심       │ 경계
  AWAITING_RESPONSE   │ 응답 대기 60초  │ 경계
  FALL_DETECTED       │ 낙상 확정       │ 긴급
  GUARDIAN_ALERTED    │ 보호자 알림 발송│ 긴급
  RESPONSE_RECEIVED   │ 응답 확인 완료  │ 복구
  RECOVERY            │ 복구 중 (8초)   │ 복구

파일 4: start.sh  (통합 실행 스크립트)
================================================================

역할: Flask 서버와 Cloudflare Tunnel을 동시에 시작합니다.
      터널 URL을 자동으로 감지해서 Flask에 등록합니다.

실행:
  ./start.sh              ← Flask + cloudflared 동시 실행
  ./start.sh --no-tunnel  ← Flask만 실행
  ./start.sh --port 8080  ← 다른 포트로 실행

동작 순서:
  1. 가상환경 활성화
  2. Flask 백그라운드 시작
  3. cloudflared 백그라운드 시작
  4. 터널 URL 자동 캡처 (최대 30초 대기)
  5. Flask /api/tunnel-url 에 URL 자동 등록
  6. URL 출력 (보호자에게 공유)
  7. Ctrl+C 시 두 프로세스 모두 종료

파일 12: ESP32_example/ESP32_dfplayer.ino  (ESP32 메인 펌웨어)
================================================================

역할: ESP32가 실행하는 메인 코드. 센서 읽기 + 데이터 전송 + 오디오 제어.
      Arduino IDE로 ESP32에 업로드합니다.

필요 라이브러리 (Arduino IDE에서 설치):
  - DFRobotDFPlayerMini
  - ArduinoJson

주요 함수:
  setup()             - WiFi 연결, DFPlayer 초기화, 핀 설정
  loop()              - 주기적으로 4가지 작업 반복

  pollAudioCommand()  - 500ms마다 Flask에서 오디오 명령 가져오기
                        GET /esp32/audio-command
  sendSensorData()    - 3000ms마다 센서 데이터를 Flask에 POST
                        ★ 현재는 더미값, 실제 센서 코드로 교체 필요
  checkButton()       - 50ms마다 아케이드 버튼 상태 확인
  postButtonPress()   - 버튼 눌림 시 Flask에 POST /button/press
  checkPlayFinished() - DFPlayer 재생 완료 감지 → 낙상 시 순차 재생
  ensureWiFi()        - WiFi 끊기면 자동 재연결

현재 더미값 위치 (실제 센서로 교체 필요한 줄):
  sendSensorData() 함수 안:
    float height = 165.0;   ← VL53L0X ToF로 교체 (센서_코드_수정_가이드.txt 참고)
    int   angle  = 88;      ← Pico UART로 교체
    bool  mmwave = true;    ← mmWave GPIO로 교체

DFPlayer SD카드 파일 구조 (/MP3 폴더):
  0001.mp3 → 평상시 배경 음악 (입실 시 자동 재생)
  0002.mp3 → 경고음 (낙상 감지 시 자동 재생)
  0003.mp3 → "낙상이 감지되었습니다." (경고음 후 자동 재생)
  0004.mp3 → "괜찮으십니까?" (낙상 의심 시 재생)
  0005.mp3 → "응답이 없으면 보호자에게..." (응답 없을 시 재생)

파일 13: ESP32_example/ESP32_setup_ap.ino  (최초 설정 펌웨어)
================================================================

역할: ESP32 처음 사용할 때만 쓰는 설정 도구.
      최초 1회 업로드 → WiFi/서버 설정 완료 → ESP32_dfplayer.ino로 교체.

동작 방식:
  1. ESP32를 "SmartBathroom_Setup" WiFi AP로 켬 (비번: setup1234)
  2. 스마트폰으로 접속 → 브라우저에서 192.168.4.1 접속

  4. 저장 → ESP32 내부 Preferences 에 저장 후 재부팅
  5. 이후 이 설정값으로 WiFi 자동 연결

※ 한 번 설정 후에는 이 파일 대신 ESP32_dfplayer.ino 사용

파일 14: ESP32_example/pico_servo.py  (Pico MicroPython)
================================================================

역할: Raspberry Pi Pico에서 실행하는 서보모터 제어 코드.
      서보를 35°~140° 왕복하며 현재 각도를 UART로 ESP32에 전송.

업로드 방법: Thonny IDE → Pico에 main.py 로 저장

동작:
  - 서보모터를 35°에서 140°까지 5° 씩 이동 (0.35초 간격)
  - 이동할 때마다 "ANGLE:35\n" 형식으로 UART 전송
  - ESP32는 이 값을 읽어서 Flask로 전달

설정 변경 위치:
  SERVO_PIN   = 15    ← 서보 연결 핀
  ANGLE_MIN   = 35    ← 스캔 시작 각도
  ANGLE_MAX   = 140   ← 스캔 끝 각도
  SCAN_STEP   = 5     ← 한 번에 몇 도씩 이동
  SCAN_DELAY  = 0.35  ← 각도마다 대기 시간 (초)
